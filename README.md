## 1. 문제 정의 (Why)

예약 시스템은 다음과 같은 질문을 반드시 마주하게 됩니다.

* 여러 사용자가 동시에 같은 자원을 예약하면 어떻게 되는가?
* DB 트랜잭션만으로 충분한가?
* 임시 예약과 확정 예약을 어떻게 구분해야 하는가?

이 프로젝트의 목적은 **이 질문들에 대한 나만의 답을 코드로 검증하는 것**입니다.

---

## 2. 프로젝트 범위 (Scope)

### 포함하려는 것

* 사용자 인증/인가 (로그인 필수)
* 단일 자원에 대한 중복 예약 방지
* 임시 예약 → 확정 예약의 상태 전이
* 동시성 상황 재현 및 테스트

### 의도적으로 제외한 것

* 실제 결제 로직
* 프론트엔드 구현
* 복잡한 추천 알고리즘

> 핵심은 **기능의 양이 아니라 문제의 깊이**입니다.

---

## 3. 현재 가설과 고민 중인 선택지

### 동시성 제어에 대한 가설

* 단순 DB 락만으로는 한계가 있을 것이다
* Redis를 활용한 락 또는 임시 상태 관리가 필요할 수 있다

→ 어떤 방식이 적절한지는 **직접 구현하고 테스트하며 결정할 예정**입니다.

### 예약 상태 설계에 대한 고민

* 예약을 한 단계로 처리할지
* 임시(TEMP) / 확정(CONFIRMED)으로 나눌지

→ 상태가 늘어날수록 복잡도가 증가하는 문제를 어떻게 관리할지 고민 중입니다.
